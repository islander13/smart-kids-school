"Is Scratch really programming, or just a game in disguise?" It's one of the most common questions from parents, often followed by: "Shouldn't we start directly with real code, like Python?" The answer comes down to one simple idea: Scratch and Python aren't two difficulty levels of the same language, they're two different tools, each with its own role.

## Scratch isn't a "baby version" of programming

Scratch, developed by MIT, is a full programming language in its own right: loops, conditions, variables, functions, events — all the fundamental concepts of programming are present. The difference is that instead of writing `if x > 10:`, you assemble a visual "if... then..." block with the mouse.

That difference isn't cosmetic. It removes one specific obstacle: syntax. An 8-year-old beginner doesn't have to worry about a forgotten semicolon or incorrect indentation — mistakes that, for an adult used to coding, are minor, but for a child can turn an entire session into a hunt for an invisible bug rather than learning logic.

## What Scratch actually teaches

With Scratch, a child learns to:
- break a problem down into steps ("first this, then that, unless...")
- spot and fix a logic error (not a syntax error)
- structure a program with loops and conditions
- think in terms of events ("when the arrow key is pressed, then...")

These are exactly the skills they'll need in Python — only "how you write it" changes. That's why in our [programme](/en#parcours), Scratch isn't a box to tick before the "real" content: it's the foundation everything else builds on.

## So, when to move on to Python?

The most reliable signal isn't age, it's comfort with Scratch. A child is ready for the transition when:

1. They build multi-block Scratch projects unaided, anticipating edge cases.
2. They start finding Scratch limiting, for example when handling lists of data or doing more complex calculations.
3. They're curious to see "what real code looks like".

With us, this transition usually goes through Python Turtle, an intermediate step that keeps an immediate visual result (drawing shapes on screen) while introducing real text syntax. It's a bridge, not a leap into the void.

## Should you "skip" Scratch if the child is older or gifted?

Rarely a good idea, even for an older child or one clearly comfortable with logic. The risk isn't that Scratch is "too easy" for them, it's that the time meant to be invested in logic gets spent fighting syntax instead — which discourages far faster than people expect, even in bright children.
